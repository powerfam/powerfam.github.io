# AI 시대 데이터 분석가 역량 (3): 스토리텔링과 커뮤니케이션

![데이터 스토리텔링](https://images.unsplash.com/photo-1551288049-bebda4e38f71?w=800&q=80)

## 분석의 90%는 커뮤니케이션이다

충격적인 사실. 대부분의 데이터 분석 결과는 실제 비즈니스 의사결정에 활용되지 않습니다.

왜일까요? 분석이 잘못되어서가 아닙니다. 의사결정자가 그 분석을 이해하지 못하거나, 설득되지 않았거나, 행동으로 옮기기 어렵다고 느꼈기 때문입니다.

AI가 아무리 정교한 분석을 해줘도, 그것을 사람에게 전달하는 것은 결국 인간의 몫입니다.

## 데이터 스토리텔링의 3요소

### 1. 데이터 (Data)
분석의 객관적 근거. 정확하고 신뢰할 수 있어야 합니다.

### 2. 비주얼 (Visual)
데이터를 한눈에 이해할 수 있게 시각화. 차트 선택이 중요합니다.

### 3. 내러티브 (Narrative)
데이터와 비주얼을 연결하는 이야기. "무엇이 일어났고, 왜 중요하며, 어떻게 해야 하는가"를 설명합니다.

## 효과적인 데이터 프레젠테이션 구조

### SCQA 프레임워크

S - Situation (상황)
"우리 회사 월간 활성 사용자가 100만 명입니다."

C - Complication (문제)
"하지만 최근 3개월간 신규 사용자 증가율이 급격히 둔화되고 있습니다."

Q - Question (질문)
"왜 이런 현상이 발생했고, 어떻게 대응해야 할까요?"

A - Answer (답변)
"분석 결과, 주요 원인은 온보딩 이탈입니다. A/B 테스트를 통해 개선된 온보딩 플로우를 제안드립니다."

## 청중별 맞춤 커뮤니케이션

### 경영진/임원

- 결론부터 말하기 (Bottom Line Up Front)
- 비즈니스 임팩트 중심 (매출, 비용, 시장점유율)
- 1페이지 요약 + 상세 자료는 별도

### 현업 담당자

- 실행 가능한 구체적 액션 아이템
- 그들의 KPI와 연결
- 제한사항과 가정 명확히

### 동료 데이터 분석가

- 방법론과 기술적 세부사항
- 데이터 소스와 한계점
- 재현 가능한 코드/쿼리 공유

## 피해야 할 실수들

1. 너무 많은 정보

슬라이드 하나에 차트 5개? 아무도 읽지 않습니다. 한 슬라이드, 한 메시지.

2. 맥락 없는 숫자

"전환율이 3%입니다" → "전환율이 3%로, 업계 평균(2%) 대비 50% 높습니다"

3. 결론 없는 분석

현상 나열로 끝나면 의미 없습니다. "그래서 제 제안은..."까지 가야 합니다.

4. 전문 용어 남발

p-value, confidence interval... 경영진은 모릅니다. 쉬운 말로 번역하세요.

## 스토리텔링 연습법

1. 엘리베이터 피치. 분석 결과를 30초 안에 설명하기
2. 할머니 테스트. 전문 지식 없는 분도 이해할 수 있는지 확인
3. So What 체인. 모든 문장에 "So What?"을 3번 물어보기
4. 피드백 루프. 실제 프레젠테이션 후 청중 반응 관찰 및 개선

데이터 분석가의 궁극적 목표는 숫자를 만드는 것이 아니라, 숫자로 조직의 행동을 바꾸는 것입니다. 스토리텔링은 그 연결고리입니다.
