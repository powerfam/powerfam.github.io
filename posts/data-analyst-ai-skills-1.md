# AI 시대 데이터 분석가 역량 (1): 프롬프트 엔지니어링

![AI 프롬프트 엔지니어링](https://images.unsplash.com/photo-1677442136019-21780ecad995?w=800&q=80)

## AI는 도구다, 그것도 아주 강력한

2024년, 데이터 분석가에게 ChatGPT나 Claude를 쓸 줄 모른다는 것은 Excel을 못 쓰는 것과 비슷해졌습니다. 하지만 같은 AI를 써도 결과물의 질은 천차만별입니다. 차이는 바로 프롬프트에 있습니다.

## 데이터 분석가를 위한 프롬프트 원칙

### 1. 역할 부여하기

```
"당신은 10년 경력의 시니어 데이터 분석가입니다.
전자상거래 회사에서 근무하며 마케팅 성과 분석을 담당합니다."
```

역할을 명시하면 AI가 해당 맥락에 맞는 답변을 생성합니다.

### 2. 구체적인 맥락 제공하기

나쁜 예. "이 데이터 분석해줘"
좋은 예. "이 CSV 파일은 최근 3개월간 고객 구매 데이터입니다. 컬럼 설명: user_id(고객ID), purchase_date(구매일), amount(금액), category(카테고리). RFM 분석을 통해 고객을 세분화해주세요."

### 3. 출력 형식 지정하기

```
결과를 다음 형식으로 정리해주세요.
1. 핵심 인사이트 (3-5개 bullet point)
2. 시각화 추천 (차트 종류와 이유)
3. 추가 분석 제안
4. Python 코드 (pandas 사용)
```

### 4. 단계별 사고 유도하기

```
"단계별로 분석해주세요.
1단계: 데이터 품질 확인
2단계: 기초 통계량 계산
3단계: 이상치 탐지
4단계: 패턴 발견
5단계: 인사이트 도출"
```

## 실전 프롬프트 템플릿

SQL 쿼리 생성.

```
다음 조건으로 SQL 쿼리를 작성해주세요.
- 테이블명: sales, customers
- 조건: 2024년 1분기, 금액 100만원 이상
- 출력: 고객별 총 구매금액 TOP 10
- 사용 DB: PostgreSQL
```

데이터 시각화 코드.

```
Python matplotlib/seaborn으로 시각화 코드를 작성해주세요.
- 데이터: 월별 매출액 (12개월)
- 차트: 라인 차트 + 전년 대비 성장률 막대 그래프
- 스타일: 비즈니스 프레젠테이션용, 깔끔한 디자인
```

## 프롬프트 반복 개선하기

한 번에 완벽한 결과를 기대하지 마세요. AI와의 대화는 반복적 개선(Iterative Refinement) 과정입니다.

1. 초기 프롬프트로 결과 확인
2. 부족한 부분 피드백. "여기서 XXX를 더 자세히 설명해줘"
3. 방향 수정. "다른 관점에서 분석해봐"
4. 최종 결과물 도출

프롬프트 엔지니어링은 기술이 아니라 AI와 효과적으로 소통하는 능력입니다. 매일 연습하면서 자신만의 프롬프트 라이브러리를 구축하세요.
